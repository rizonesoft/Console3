# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 1.x     | :white_check_mark: |

## Reporting a Vulnerability

If you discover a security vulnerability in Console3, please report it responsibly:

1. **Do NOT** open a public GitHub issue
2. Email security concerns to: security@rizonesoft.com
3. Include:
   - Description of the vulnerability
   - Steps to reproduce
   - Potential impact
   - Any suggested fixes (if applicable)

## Response Timeline

- **Acknowledgment**: Within 48 hours
- **Initial Assessment**: Within 1 week
- **Resolution Target**: Within 30 days for critical issues

## Security Best Practices

When using Console3:

- Keep Console3 updated to the latest version
- Only run trusted shell commands
- Be cautious with scripts from untrusted sources
- Review any custom configuration files

## Scope

This security policy applies to:

- Console3 application
- Official Console3 releases
- Console3 installer packages

Third-party dependencies (WTL, WIL, libvterm) should be reported to their respective maintainers.
